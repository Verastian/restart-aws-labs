# 💻 Línea de comandos de Linux

<IconEC2 title="Amazon EC2" width="72" height="72" />

⏱️ **Tiempo estimado:** 30 minutos &nbsp;|&nbsp; 📊 **Nivel:** Básico &nbsp;|&nbsp; 🏷️ **Servicio:** Amazon EC2

En este laboratorio se ejecutan comandos de Linux dentro de una instancia Amazon EC2 para obtener información sobre el sistema y la sesión activa, y se practican técnicas de búsqueda y reutilización del historial de comandos Bash.

> **Nota:** Este laboratorio depende del material didáctico de cursos anteriores. Se recomienda haber completado el Lab 225 antes de continuar.

---

## 🎯 Temas que se abordarán

Al finalizar este laboratorio, podrá:

- ✅ Conectarse a una instancia EC2 de Amazon Linux vía SSH
- ✅ Ejecutar comandos para obtener información del sistema y la sesión activa
- ✅ Buscar y reutilizar comandos anteriores del historial de Bash

Este laboratorio utiliza los siguientes componentes de AWS:

- 🖥️ **Amazon EC2 — Host de comandos** (en subred pública): instancia de Amazon Linux donde se ejecutan todos los comandos del ejercicio.

## 💡 Conceptos clave

:::info[🐚 Bash — Bourne Again Shell]
Intérprete de comandos predeterminado en la mayoría de distribuciones Linux, incluyendo Amazon Linux. Bash ofrece funciones avanzadas como autocompletado con Tab, historial de comandos (CTRL+R), variables de entorno y scripting.
:::

:::info[📜 Historial de comandos en Bash]
Bash almacena automáticamente todos los comandos ejecutados en la sesión. El comando `history` lista el historial completo. CTRL+R permite buscar comandos previos en tiempo real. El símbolo `!!` repite el último comando ejecutado.
:::

:::info[🌍 Variables de entorno en Linux]
Las variables de entorno configuran el comportamiento del sistema y los programas. `TZ` define la zona horaria para los comandos de fecha/hora. Se pueden asignar temporalmente antes de un comando: `TZ=America/New_York date`.
:::

:::info[📅 Calendario juliano]
El formato de fecha juliana usa una numeración continua de días dentro del año, sin reiniciar al cambiar de mes. Por ejemplo, el 1 de febrero es el día 32 del año. El comando `cal -j` muestra el calendario en este formato.
:::

## Tarea 2: Ejecutar comandos conocidos

En este ejercicio se ejecutan comandos para obtener información general sobre el sistema y la sesión activa.

### Paso 1 — Identificar el usuario actual (`whoami`)

Introduzca `whoa` y presione **Tab**. Bash completará automáticamente el comando a `whoami`. Presione **Enter**:

```bash
whoami
```

![](imgs/Pasted%20image%2020260606200047.png)

### Paso 2 — Ver el nombre del equipo (`hostname`)

Introduzca `hostname -s` y presione **Enter** para ver la versión abreviada del nombre del servidor:

```bash
hostname -s
```

![](imgs/Pasted%20image%2020260606200329.png)

### Paso 3 — Ver el tiempo de actividad (`uptime`)

Introduzca `uptime -p` y presione **Enter** para ver cuánto tiempo lleva corriendo el sistema en formato legible:

```bash
uptime -p
```

![](imgs/Pasted%20image%2020260606200354.png)

![](imgs/Pasted%20image%2020260606191850.png)

*Los comandos `whoami`, `hostname -s` y `uptime -p` proporcionan información básica sobre el sistema. Útiles para diagnóstico y solución de problemas.*

### Paso 4 — Ver información de usuarios conectados (`who`)

Introduzca `who -H -a` y presione **Enter** para ver los usuarios conectados y datos adicionales de la sesión:

```bash
who -H -a
```

![](imgs/Pasted%20image%2020260606200428.png)

![](imgs/Pasted%20image%2020260606191956.png)

*El comando `who -H -a` muestra: nombre de usuario, línea de terminal, hora de inicio de sesión, tiempo inactivo, PID del proceso y comentarios.*

### Paso 5 — Mostrar fecha y hora en diferentes zonas horarias (`TZ=`)

Introduzca los siguientes comandos para ver la fecha y hora en Nueva York y Los Ángeles:

```bash
TZ=America/New_York date
```

```bash
TZ=America/Los_Angeles date
```

![](imgs/Pasted%20image%2020260606200527.png)

![](imgs/Pasted%20image%2020260606192043.png)

*La variable de entorno `TZ` define la zona horaria para el comando `date`. En este ejemplo se muestra la fecha y hora en formato: día, mes, fecha, hora, zona horaria y año.*

:::note
Si la hora configurada en el sistema no es correcta, la información de zona horaria calculada también será incorrecta.
:::

### Paso 6 — Mostrar el calendario con fechas julianas (`cal -j`)

El formato juliano usa una secuencia continua de días sin reiniciar al cambiar de mes (el 1 de febrero es el día 32 del año). Ingrese:

```bash
cal -j
```

![](imgs/Pasted%20image%2020260606200903.png)

![](imgs/Pasted%20image%2020260606192122.png)

*El comando `cal -j` muestra el mes actual en formato de fecha juliana. Por ejemplo, el 24 de septiembre corresponde al día 245 del año.*

### Paso 7 — Ver diferentes vistas del calendario

```bash
cal -s
```

```bash
cal -m
```

![](imgs/Pasted%20image%2020260606200952.png)

![](imgs/Pasted%20image%2020260606192206.png)

*`cal -s` muestra el calendario con el domingo como primer día de la semana. `cal -m` lo muestra empezando por el lunes.*

:::note
Existen muchas formas de visualizar los calendarios. Consulte `man cal` para más opciones.
:::

### Paso 8 — Ver el ID de usuario y grupos (`id`)

Ingrese `id ec2-user` y presione **Enter** para ver el ID único y la información de grupos del usuario actual:

```bash
id ec2-user
```

![](imgs/Pasted%20image%2020260606201031.png)

![](imgs/Pasted%20image%2020260606192251.png)

*La salida de `id ec2-user` muestra el UID (ID de usuario), GID (ID del grupo principal) y los grupos a los que pertenece el usuario.*

## Tarea 3: Mejorar el flujo de trabajo con el historial de comandos

En esta tarea se reduce la carga de trabajo reutilizando comandos anteriores mediante el historial de Bash.

### Paso 1 — Ver el historial de comandos

Escriba `history` y presione **Enter** para ver todos los comandos ejecutados en esta sesión:

```bash
history
```

![](imgs/Pasted%20image%2020260606201105.png)

![](imgs/Pasted%20image%2020260606192331.png)

*Al ejecutar `history` se muestra la lista numerada de todos los comandos utilizados en el laboratorio.*

### Paso 2 — Buscar en el historial inverso (CTRL+R)

Presione **CTRL+R** para activar la búsqueda en el historial inverso. Luego introduzca `TZ` y presione **Tab**. Aparecerá un comando de fecha anterior que puede editar con las teclas de dirección:

```
(reverse-i-search)`TZ': TZ=America/New_York date
```

![](imgs/Pasted%20image%2020260606201311.png)

![](imgs/Pasted%20image%2020260606201350.png)

![](imgs/Pasted%20image%2020260606192356.png)

*CTRL+R activa la búsqueda en el historial. Al introducir `TZ` y presionar Tab, Bash encuentra y muestra el comando de fecha anterior listo para editar y ejecutar.*

:::note
La búsqueda inversa con CTRL+R permite editar los comandos encontrados. Use las teclas de dirección para navegar entre resultados.
:::

### Paso 3 — Repetir el último comando (`!!`)

Introduzca `date` y presione **Enter**. Luego introduzca `!!` y presione **Enter** nuevamente para ejecutar el mismo comando:

```bash
date
```

```bash
!!
```

![](imgs/Pasted%20image%2020260606201452.png)

![](imgs/Pasted%20image%2020260606192416.png)

*El símbolo `!!` es un atajo para ejecutar el último comando introducido. En este ejemplo, `date` se ejecutó dos veces usando `!!`.*

<summary>📋 Ver respuestas</summary>

| # | Respuesta | Explicación |
|---|-----------|-------------|
| 1 | **B) `whoami`** | `whoami` imprime el nombre del usuario actual. `id` también lo muestra pero con información adicional del UID y grupos. |
| 2 | **B) `uptime -p`** | La opción `-p` (pretty) formatea la salida como "up X hours, Y minutes". Sin `-p`, la salida incluye la hora actual y carga del sistema. |
| 3 | **C) CTRL+R** | CTRL+R activa el modo de búsqueda inversa en el historial de Bash. Al escribir caracteres, Bash busca el comando más reciente que los contenga. |
| 4 | **C) Repite el último comando** | `!!` es un alias del último comando ejecutado. Es equivalente a presionar ↑ + Enter. Útil para repetir comandos largos o añadir `sudo !!`. |

| 8 | **C) Lista numerada de comandos** | `history` muestra todos los comandos almacenados en el historial, con un número de línea que puede usarse para ejecutarlos con `!<número>`. |
